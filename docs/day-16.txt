OVERVIEW:
Today's implementation focused on Phase 5: "The Missions". We built a cinematic,
high-performance Projects section that breaks the traditional vertical flow
using horizontal scroll pinning and interactive mission briefings.

--------------------------------------------------------------------------------
1. KEY FEATURES IMPLEMENTED
--------------------------------------------------------------------------------

A. HORIZONTAL SCROLL ENGINE (GSAP ScrollTrigger)
   - The entire section pins to the viewport when it reaches the top.
   - Vertical mouse scroll is mapped to horizontal X-translation of the projects container.
   - Recalculates scroll bounds dynamically on window resize or when filters change.

B. DYNAMIC CATEGORY FILTERING (GSAP Flip)
   - Projects can be filtered by category (Fullstack, Frontend, AI, Backend).
   - Used GSAP's "Flip" plugin to animate the layout change smoothly, preventing 
     abrupt jumps when items enter or exit the filtered list.

C. MISSION BRIEFING MODAL
   - A full-screen, Stark-Industries-style modal for "Deep Dive" briefings.
   - Features structured content: Problem Statement, Solution, and Operational Impact.
   - Cinematic entry/exit animations using Framer Motion (AnimatePresence).

D. THEMATIC BRANDING & ASSETS
   - Integrated custom SVG icons for GitHub to circumvent library limitations.
   - Implemented "MISSION_ID" nomenclature and coding-style status badges.
   - Added background HUD overlays with rotating text for depth.

--------------------------------------------------------------------------------
2. TECHNICAL CHALLENGES & SOLUTIONS
--------------------------------------------------------------------------------

CHALLENGE: Recalculating Scroll Bounds
When a user filters projects, the width of the horizontal container changes. 
If the scroll bounds are not updated, the user might scroll into empty space 
or the section might unpin too early.
SOLUTION: We use a `useEffect` hook that watches the `filteredProjects` state. 
When it changes, we kill the previous ScrollTrigger instance and initialize 
a new one with the fresh dimensions of the container.

CHALLENGE: Lucide-React Icon Missing
The version of `lucide-react` installed (1.9.0) does not include brand icons 
like GitHub.
SOLUTION: Instead of adding a heavy external dependency, we implemented a 
lightweight custom SVG component for GitHub, ensuring the design remains premium 
and dependency-light.

--------------------------------------------------------------------------------
3. INTERVIEW PREPARATION (Q&A)
--------------------------------------------------------------------------------

Q: How do you handle horizontal scrolling in a vertical layout?
A: I use GSAP ScrollTrigger's "pin" feature. By pinning the parent section in 
place and calculating the difference between the container width and the 
viewport width, I can translate the inner container along the X-axis while 
the user scrolls vertically. This creates a "track" effect where vertical 
effort translates into horizontal progress.

Q: Why use GSAP Flip instead of standard CSS transitions for filtering?
A: Standard CSS transitions (like opacity or scale) don't handle layout flow 
well—elements usually "jump" to their new positions. Flip (First, Last, Invert, 
Play) records the initial positions, updates the state, and then calculates the 
delta to animate elements from their old to new coordinates seamlessly.

Q: How do you optimize large images in a scroll-intensive section?
A: I use Next.js's `next/image` component with the `fill` property and 
`priority` (where needed). For the Project cards, we use `object-cover` and 
`will-change: transform` to move the heavy lifting to the GPU, ensuring 
60fps performance even during complex scroll transitions.

Q: What is the benefit of the GSAP Context API in React?
A: React's Strict Mode and fast refresh can cause animations to double-trigger 
or leak memory. `gsap.context()` provides a wrapper that collects all 
animations and ScrollTriggers created within it, allowing us to clean up 
everything in a single `revert()` call during the component unmount.
