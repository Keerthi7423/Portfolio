────────────────────────────────────────────────────────────────────────────────
  GSAP + ScrollTrigger
────────────────────────────────────────────────────────────────────────────────

  Package       : gsap
  Version       : 3.x
  Cost          : Free (GSAP 3 core + ScrollTrigger = free for all use cases)

  NOTE: GSAP Club plugins (SplitText, MorphSVG) are paid.
  This project uses ONLY free plugins.

  FREE GSAP PLUGINS USED:
  → ScrollTrigger   — scroll-driven animations (all section reveals)
  → ScrollToPlugin  — smooth scroll-to on nav link click
  → Flip            — layout transition animations (project filter)
  → Observer        — custom scroll detection for horizontal section

  WHAT GSAP HANDLES:
  → Loading screen letter-by-letter name assembly
  → Hero text staggered entrance (each word/line separate timeline)
  → ALL section scroll-trigger entrances (8 unique entrance signatures)
  → Horizontal scroll conversion for Projects/Missions section
       gsap.to(projectsTrack, {
         xPercent: -100 * (cards - 1),
         ease: "none",
         scrollTrigger: {
           trigger: projectsSection,
           pin: true,
           scrub: 1,
           snap: 1 / (cards - 1),
         }
       })
  → Timeline spine draw (stroke-dashoffset animation, Experience section)
  → Stat counter animations (0 → final number, 1.5s ease-out)
  → Page scroll progress bar (top of page, width tracks scroll %)
  → Background color transitions between sections (onEnter/onLeave)

  GSAP SETUP PATTERN:
  → All GSAP code lives in useEffect() hooks
  → Always return cleanup: return () => ctx.revert()
  → Use gsap.context() for scoped animations
  → Register plugins once in app/layout.js:
    gsap.registerPlugin(ScrollTrigger, ScrollToPlugin, Flip, Observer)

────────────────────────────────────────────────────────────────────────────────
  FRAMER MOTION
────────────────────────────────────────────────────────────────────────────────

  Package       : framer-motion
  Version       : 11.x
  Cost          : Free, MIT license

  WHAT FRAMER MOTION HANDLES (complement to GSAP, not replacement):
  → Component mount/unmount animations (JARVIS panel open/close)
  → Tech card 3D flip (rotateY on hover)
  → Project modal entrance/exit (scale + opacity)
  → Page-level layout animations
  → Stagger animations on lists (variants + staggerChildren)
  → Testimonial card scan-line reveal

  RULE: GSAP owns scroll-driven and timeline animations.
        Framer Motion owns component-level state-driven animations.
        Never use both on the same element.

────────────────────────────────────────────────────────────────────────────────
  LENIS (Smooth Scroll)
────────────────────────────────────────────────────────────────────────────────

  Package       : @studio-freight/lenis
  Version       : 1.x
  Cost          : Free, MIT license

  WHY LENIS:
  → Native browser scroll feels choppy compared to inertia-based scroll
  → Lenis adds momentum/easing to scroll without breaking ScrollTrigger
  → Required for the cinematic feel of the overall scroll journey

  SETUP:
  → Initialize in app/layout.js (or a root Provider component)
  → Connect Lenis RAF (requestAnimationFrame) to GSAP ticker:
    gsap.ticker.add((time) => lenis.raf(time * 1000))
    gsap.ticker.lagSmoothing(0)
  → Disable on mobile (native touch scroll is already smooth)

────────────────────────────────────────────────────────────────────────────────
  GEMINI API (Google AI)
────────────────────────────────────────────────────────────────────────────────

  Service       : Google AI Studio — Gemini 1.5 Flash
  Cost          : FREE TIER
  Free limits   : 15 RPM (requests/minute), 1M tokens/day, 1500 RPD
  API key       : Get at aistudio.google.com (no credit card needed)
  SDK           : @google/generative-ai (official npm package)

  WHY GEMINI 1.5 FLASH:
  → Fastest Gemini model — low latency for chat responses
  → 1M token context window — entire knowledge base fits easily
  → Free tier is genuinely generous for a portfolio
  → Streaming support (generateContentStream) for live typewriter effect

  GEMINI API ROUTE (app/api/jarvis/route.js):

  import { GoogleGenerativeAI } from '@google/generative-ai'

  const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY)

  export async function POST(request) {
    const { messages, userMessage } = await request.json()

    const model = genAI.getGenerativeModel({
      model: 'gemini-1.5-flash',
      systemInstruction: JARVIS_SYSTEM_PROMPT,  // developer knowledge base
    })

    const chat = model.startChat({ history: messages })

    // Streaming response
    const result = await chat.sendMessageStream(userMessage)

    const stream = new ReadableStream({
      async start(controller) {
        for await (const chunk of result.stream) {
          const text = chunk.text()
          controller.enqueue(new TextEncoder().encode(text))
        }
        controller.close()
      }
    })

    return new Response(stream, {
      headers: { 'Content-Type': 'text/plain; charset=utf-8' }
    })
  }

  RATE LIMITING (free, no extra package):
  → Simple in-memory Map: IP → { count, resetTime }
  → Max 20 requests per IP per hour
  → Returns 429 with "JARVIS is resting. Try again shortly." message

  SYSTEM PROMPT STRUCTURE:
  → Lives in lib/jarvis-knowledge.js as a JS template literal
  → Sections: identity, personality, tech stack, projects, availability,
              rates, FAQ, contact info, work philosophy
  → Total tokens: ~2000–4000 (well within limits)
  → Instructs model to: be concise (max 3 paragraphs), stay in character,
    end with subtle CTA, say "contact directly" if unsure

────────────────────────────────────────────────────────────────────────────────
  TEXT-TO-SPEECH — Web Speech API (SpeechSynthesis)
────────────────────────────────────────────────────────────────────────────────

  API           : window.speechSynthesis (browser built-in)
  Package       : None — zero dependencies
  Cost          : Free forever
  Support       : Chrome ✓, Edge ✓, Safari ✓, Firefox partial

  IMPLEMENTATION (hooks/useSpeech.js):

  export function useSpeech() {
    const speak = (text, options = {}) => {
      window.speechSynthesis.cancel()
      const utterance = new SpeechSynthesisUtterance(text)
      utterance.voice = getBestVoice()   // prefer deep male English voice
      utterance.rate  = options.rate  || 0.92   // slightly slower = JARVIS feel
      utterance.pitch = options.pitch || 0.85   // slightly lower pitch
      utterance.volume = options.volume || 1.0
      window.speechSynthesis.speak(utterance)
      return utterance
    }

    const getBestVoice = () => {
      const voices = window.speechSynthesis.getVoices()
      // Prefer: Google UK English Male > Microsoft David > any English male
      return voices.find(v => v.name.includes('Google UK English Male'))
          || voices.find(v => v.name.includes('David'))
          || voices.find(v => v.lang === 'en-GB')
          || voices[0]
    }

    const stop = () => window.speechSynthesis.cancel()

    return { speak, stop }
  }

  NOTES:
  → Always call cancel() before new utterance (prevents queue build-up)
  → getVoices() is async on first load — use onvoiceschanged event
  → Mute state stored in React context, checked before every speak() call
  → On iOS Safari: utterance must be triggered by direct user interaction

────────────────────────────────────────────────────────────────────────────────
  SPEECH-TO-TEXT — Web Speech API (SpeechRecognition)
────────────────────────────────────────────────────────────────────────────────

  API           : window.SpeechRecognition (browser built-in)
  Package       : None — zero dependencies
  Cost          : Free forever
  Support       : Chrome ✓, Edge ✓ | Safari/Firefox: fallback to text input

  IMPLEMENTATION (hooks/useSpeechRecognition.js):

  export function useSpeechRecognition({ onResult, onEnd }) {
    const SpeechRecognition =
      window.SpeechRecognition || window.webkitSpeechRecognition

    if (!SpeechRecognition) return { supported: false }

    const recognition = new SpeechRecognition()
    recognition.continuous    = false
    recognition.interimResults = true
    recognition.lang           = 'en-US'

    recognition.onresult = (event) => {
      const transcript = Array.from(event.results)
        .map(r => r[0].transcript)
        .join('')
      onResult(transcript, event.results[0].isFinal)
    }

    recognition.onend = onEnd

    return {
      supported: true,
      start: () => recognition.start(),
      stop:  () => recognition.stop(),
    }
  }

────────────────────────────────────────────────────────────────────────────────
  AUDIO VISUALIZER — Web Audio API
────────────────────────────────────────────────────────────────────────────────

  API           : window.AudioContext (browser built-in)
  Package       : None — zero dependencies
  Cost          : Free forever

  HOW IT WORKS:
  → When SpeechSynthesis starts speaking, connect utterance audio to
    an AnalyserNode via AudioContext
  → AnalyserNode.getByteFrequencyData() returns FFT frequency array
  → requestAnimationFrame loop reads this array and draws bars on Canvas

  NOTE: SpeechSynthesis ↔ AudioContext connection is tricky cross-browser.
  Fallback: if AudioContext connection fails, animate bars with a simulated
  sine-wave pattern synced to utterance duration. Visually identical.

  CANVAS VISUALIZER DESIGN:
  → Full width of JARVIS panel, 40px tall
  → Bars: arc-blue (#4fc3f7), centered vertically (mirror up + down)
  → Bar count: 48 bars
  → Bar gap: 2px
  → Peak bars: white highlight on tallest bars
  → Fades in when speaking, fades out on silence

  API ROUTE (app/api/contact/route.js):

  import { Resend } from 'resend'

  const resend = new Resend(process.env.RESEND_API_KEY)

  export async function POST(request) {
    const { name, email, subject, message } = await request.json()

    // Basic validation
    if (!name || !email || !message) {
      return Response.json({ error: 'Missing fields' }, { status: 400 })
    }

    await resend.emails.send({
      from:    'JARVIS <noreply@yourdomain.com>',
      to:      'you@youremail.com',
      subject: `[MISSION BRIEF] ${subject} — from ${name}`,
      html:    buildEmailTemplate({ name, email, subject, message }),
    })

    return Response.json({ success: true })
  }

  13. API ARCHITECTURE
================================================================================

  CLIENT (Browser)
      │
      │  fetch('/api/jarvis', { method: 'POST', body: { message, history } })
      ▼
  NEXT.JS API ROUTE (Vercel Serverless Function)
      │
      │  Validates request + checks rate limit
      │  Injects developer knowledge base as system prompt
      │  Calls Gemini 1.5 Flash API with streaming
      │
      │  ReadableStream → piped back to client
      ▼
  CLIENT (Browser)
      │
      │  Reads stream chunk by chunk
      │  Updates chat UI with typewriter effect
      │  When stream ends, triggers Web Speech API TTS
      ▼
  WEB SPEECH API
      │
      │  Speaks the full response text
      │  Fires onstart → AudioVisualizer activates
      │  Fires onend → AudioVisualizer deactivates
      ▼
  USER HEARS JARVIS RESPOND

  ─────────────────────────────────────────────────────

  CONTACT FORM FLOW:

  CLIENT
    → POST /api/contact { name, email, subject, message }
    → Shows loading state

  API ROUTE
    → Validates fields
    → Calls Resend API
    → Returns { success: true }

  CLIENT
    → Shows success animation
    → Resets form
