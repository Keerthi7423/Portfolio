--------------------------------------------------------------------------------
1. TECHNICAL IMPLEMENTATION DETAILS
--------------------------------------------------------------------------------

A. 3D Flip Mechanics (SkillCard.jsx)
   - Used CSS Perspective: Applied `perspective: 1000px` to the parent container to
     create a realistic sense of depth when cards rotate.
   - Preserve-3D: Set `transform-style: preserve-3d` on the motion wrapper to ensure
     children (Front & Back faces) maintain their 3D positions.
   - Backface-Visibility: Used `backface-visibility: hidden` to hide the "reverse side"
     of a face when it's flipped away from the viewer.
   - Framer Motion Integration: Controlled the rotation state via `rotateY` animate
     props tied to `onMouseEnter` and `onMouseLeave` events.

B. Cinematic "Fly-In" Staggers (SkillsSection.jsx)
   - Staggered Entry: Used `containerVariants` with `staggerChildren` to reveal
     cards one by one when the section enters the viewport.
   - 3D Entry Animation: Each card "flies in" using a combination of:
     - `y: 100` (starts below)
     - `x: 30` (slight side entry)
     - `rotateY: 45` & `rotateZ: 5` (entering at an angle)
     - `opacity: 0` to `1`
   - Ease-Out Expo: Used a custom cubic-bezier `[0.16, 1, 0.3, 1]` for a premium, 
     snappy-yet-smooth "Stark-style" motion feel.

C. Visual Polish
   - Holographic Glare: Added a gradient sweep that moves across the card on hover.
   - Scanner Line: Added a continuous vertical scanning animation (`animate-scan-y`).
   - Power Bar: Implemented a dynamic progress bar on the back of the card that 
     animates only when the card is flipped.

--------------------------------------------------------------------------------
2. INTERVIEW PREPARATION (Q&A)
--------------------------------------------------------------------------------

Q1: How do you implement a 3D flip effect in CSS/React?
A: You need three main layers:
   1. Perspective Container: Defines the viewing distance (e.g., `perspective: 1000px`).
   2. 3D Wrapper: Has `transform-style: preserve-3d` and handles the rotation.
   3. Two Faces (Front/Back): Positioned absolutely within the wrapper. One is 
      rotated 180deg by default. Both must have `backface-visibility: hidden`.

Q2: Why use rotateY for 3D instead of just a 2D flip?
A: `rotateY` uses the GPU for rendering (hardware acceleration), which is much 
   smoother than animating 2D properties like `width` or `left`. It also adds 
   realism by simulating a 3D space.

Q3: How does 'staggerChildren' work in Framer Motion?
A: It's a property in the `transition` object of a parent variant. When the parent
   switches to its 'visible' state, `staggerChildren` ensures that children using
   the same variant names will trigger their animations with a specified delay 
   relative to each other (e.g., 0.1s delay between each card).

Q4: What is 'Viewport' in Framer Motion and why use it?
A: The `whileInView` prop and `viewport` config allow animations to trigger 
   automatically when an element enters the browser's visible area. We use it to 
   ensure the "Superpowers" fly in only when the user scrolls to that section, 
   saving resources and creating a better narrative flow.

Q5: How do you ensure accessibility for interactive cards that flip?
A: You should ensure that users can flip cards using keyboard (Tab + Enter/Space) 
   and that screen readers can read both sides. Using `aria-live` or managing 
   `aria-hidden` attributes correctly is key. (Note: Current implementation focus
   is visual; production would need tabIndex and keyboard listeners).
