OVERVIEW
--------------------------------------------------------------------------------
Day 24 focused on bringing JARVIS to life visually. Building upon the API established 
in Day 23, we implemented the "Body" of the AI — a retractable HUD (Heads-Up Display) 
panel and an interactive "Arc Reactor" trigger. The goal was to create an immersive, 
cinematic experience that feels like a real Stark Industries interface.

KEY FEATURES IMPLEMENTED
--------------------------------------------------------------------------------
1.  ARC REACTOR TRIGGER (JarvisTrigger.jsx):
    - A floating action button (FAB) styled as Tony Stark's Arc Reactor.
    - Features infinite pulse animations and rotating geometric rings.
    - Uses CSS/Tailwind gradients and box-shadows to create a "glowing" effect.
    - Transitions color from "Arc Blue" to "Marvel Red" when the panel is active.

2.  RETRACTABLE HUD PANEL (JarvisChatPanel.jsx):
    - A sliding glassmorphism panel with corner bracket aesthetics.
    - Integrated with `framer-motion` for smooth spring-based entry/exit animations.
    - Features a scrollable message area with unique styles for User vs. JARVIS.
    - Implemented a "HUD Overlay" feel using scanlines and noise textures.

3.  CHAT HOOK (useJarvis.js):
    - A custom React hook to manage chat history and loading states.
    - Handles the communication with the `/api/jarvis` endpoint.
    - Automatically manages "multi-turn" context by passing history back to the AI.

4.  GLOBAL INTEGRATION:
    - Injected the `JarvisManager` into the `RootLayout`, making JARVIS available 
      on every page of the portfolio without re-initializing.

TECHNICAL DEEP-DIVE (INTERVIEW PREP)
--------------------------------------------------------------------------------

Q1: How did you achieve the "Stark-style" UI look without using heavy images?
A: 
   - Layered Borders: Using `border-t-2 border-l-2` on absolute-positioned divs 
     to create "corner brackets" instead of full boxes.
   - Glassmorphism: Combining `bg-void/90` with `backdrop-blur-xl` and 
     `border-arc-blue/30` to create a semi-transparent, futuristic feel.
   - Micro-Animations: Using `animate-pulse` and `animate-spin-slow` on small 
     elements (like status dots or reactor rings) to make the UI feel "alive."

Q2: Why use a custom hook (useJarvis) instead of putting logic in the component?
A: Separation of Concerns. 
   - The UI component (`JarvisChatPanel`) should only care about how to display 
     data and handle user input events.
   - The logic for API calls, state management, and error handling belongs in a 
     hook. This makes the code more testable and allows us to reuse the JARVIS 
     logic in other parts of the app (e.g., if we wanted a separate voice-only 
     trigger).

Q3: Explain the animation strategy used for the HUD panel.
A: I used `framer-motion`'s `AnimatePresence` and `motion.div`. 
   - `initial`: Sets the starting state (scale 0.9, hidden, offset).
   - `animate`: The active state (scale 1, visible, original position).
   - `exit`: The state when the component is unmounted.
   Using a `spring` transition (`damping: 25, stiffness: 300`) gives it a 
   mechanical, high-quality feel that "bounces" into place rather than a 
   simple fade.

Q4: How do you handle the "Auto-scroll to bottom" in a chat interface?
A: I used a combination of `useRef` and `useEffect`. 
   - An empty div (`messagesEndRef`) is placed at the bottom of the message list.
   - Every time the `messages` array or `isLoading` state changes, a `useEffect` 
     calls `scrollIntoView({ behavior: "smooth" })` on that ref.
   - This ensures the user always sees the latest response without manual scrolling.
