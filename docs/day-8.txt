OVERVIEW:
Today's mission was to transition from the 2D "Cinematic Shell" into the 3D 
"Universe." We initialized the Three.js foundation using React Three Fiber (R3F) 
and implemented a high-density particle system that serves as the immersive 
backdrop for the Hero section.

--------------------------------------------------------------------------------
1. TECHNICAL ARCHITECTURE: THE 3D ENGINE (R3F)
--------------------------------------------------------------------------------

- WHY REACT THREE FIBER?
  - Three.js is a powerful imperative library, but managing it in a declarative 
    framework like React can be messy. R3F allows us to build 3D scenes using 
    reusable components, making the "Universe" easier to scale and maintain.
  - It handles the animation loop, scene graph, and state management 
    automatically via the `<Canvas />` component.

- IMPLEMENTATION: HeroScene.jsx
  - The Scene Graph: Configured with a `<PerspectiveCamera />` for depth, 
    `<AmbientLight />` for base visibility, and strategic `<PointLight />` 
    sources to create the "Arc Blue" and "Marvel Red" cinematic contrast.
  - Performance Optimization: 
    - `dpr={[1, 2]}`: Limits the device pixel ratio to 2 to prevent performance 
      tanking on 4K/Retina displays.
    - `powerPreference: "high-performance"`: Signals the browser to use the 
      dedicated GPU if available.
    - `Suspense`: Ensures the 3D scene only renders when all assets/geometries 
      are ready, preventing "popping" artifacts.

--------------------------------------------------------------------------------
2. COMPONENT: THE NEURAL PARTICLE FIELD
--------------------------------------------------------------------------------

- LOGIC: ParticleField.jsx (Task 3.2)
  - Data Structure: Used `Float32Array` to store 7,500 coordinates (2,500 points 
    * 3 axes). This is passed directly to a `bufferAttribute` for maximum 
    GPU efficiency.
  - Physics & Animation:
    - Drift: Implemented a gentle rotation in the `useFrame` loop using 
      `clock.getElapsedTime()`.
    - Mouse Parallax: Used `THREE.MathUtils.lerp` to smoothly transition the 
      entire particle field based on the user's mouse position. This creates 
      a "holographic" feeling where the UI reacts to the user's focus.
    - Material: Used `AdditiveBlending` and `transparent: true` to make the 
      particles glow when they overlap, mimicking energy particles.

--------------------------------------------------------------------------------
3. INTERVIEW PREPARATION: "HOW WOULD YOU EXPLAIN THIS?"
--------------------------------------------------------------------------------

Q: Why use `useMemo` for the particle positions?
A: Generating 2,500 random positions is computationally expensive. If we did 
   this inside the component body, it would re-run on every render. `useMemo` 
   ensures we only calculate those positions once when the component mounts.

Q: What is "Lerping" and why use it for mouse movement?
A: LERP stands for Linear Interpolation. Instead of setting the position 
   instantly (which looks jittery), we move a fraction of the distance towards 
   the target every frame. This creates the "weighted," smooth follow effect 
   that feels premium.

Q: How do you handle 3D on mobile devices?
A: We implemented a "Responsive Viewport Detector." If the screen width is less 
   than 768px, we bypass the R3F Canvas entirely and render a high-performance 
   CSS fallback (noise + gradients). This ensures the "Hero" looks great even 
   on devices without powerful GPUs.

Q: What is the purpose of `depthWrite={false}` in the particle material?
A: Particles are transparent. If `depthWrite` was true, particles in front 
   would "cut out" particles behind them because the depth buffer thinks 
   something is already there. Setting it to false allows us to see through 
   overlapping particles, creating a beautiful glowing additive effect.
