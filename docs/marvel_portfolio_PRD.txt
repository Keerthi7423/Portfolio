                  collaborators, and startups seeking development talent
Core Vision     : An immersive, cinematic portfolio experience inspired by the

--------------------------------------------------------------------------------
2. PROBLEM STATEMENT
--------------------------------------------------------------------------------

- Standard portfolios are flat, forgettable, and fail to communicate the
  developer's personality, depth, and energy.
- Clients and recruiters skim dozens of portfolios — there is no differentiation.
- No existing portfolio format provides interactive, conversational access to
  the developer's capabilities 24/7.
- There is no emotional journey — no story arc that draws the visitor in.


--------------------------------------------------------------------------------
3. GOALS & SUCCESS METRICS
--------------------------------------------------------------------------------

GOALS:
  [G1] Create a visually cinematic, 3D scrolling experience that is unforgettable.
  [G2] Guide visitors through a story — from Hero intro → Powers → Missions →
       Testimonials → Contact — using scroll-driven motion.
  [G3] Deploy an AI Voice Agent (JARVIS-style) that answers questions about
       the developer 24/7, replacing a static FAQ or contact form.
  [G4] Demonstrate the developer's own engineering skill through the portfolio
       itself (meta-proof of capability).
  [G5] Convert visitors into leads — drive contact form fills, calls, or hires.

SUCCESS METRICS:
  - Average session duration > 3 minutes
  - Scroll depth > 80% (visitors reach the Contact section)
  - AI Agent interaction rate > 40% of visitors
  - Contact form conversion rate > 8%
  - Mobile performance score > 85 (Lighthouse)
  - Desktop performance score > 90 (Lighthouse)


--------------------------------------------------------------------------------
4. TARGET USERS
--------------------------------------------------------------------------------

PRIMARY USERS:
  - Startup founders evaluating full stack developers for hire / contract
  - Tech recruiters at product companies
  - Product managers looking for a technical co-founder or contractor

SECONDARY USERS:
  - Fellow developers exploring inspiration or collaboration
  - Design/tech community sharing on social media (virality driver)

USER GOALS:
  - Quickly understand what the developer builds and their tech stack
  - Feel confidence in the developer's capabilities and personality
  - Ask specific questions without waiting for an email reply
  - Find a way to reach out or book a call

7. DESIGN SYSTEM
--------------------------------------------------------------------------------

COLOR PALETTE:
  Primary Background  : #0a0a0f  (near-black, deep space)
  Secondary BG        : #0d1117  (dark blue-black)
  Marvel Red          : #e23636  (hero accent)
  Gold / Stark        : #c8a951  (premium accent)
  Arc Blue            : #4fc3f7  (JARVIS / tech accent)
  Text Primary        : #f0f0f0  (near white)
  Text Secondary      : #8b8fa8  (muted)
  Surface             : #13151e  (card backgrounds)
  Border              : #1e2235  (subtle dividers)

TYPOGRAPHY:
  Display / Hero      : "Bebas Neue" or "Cinzel Decorative" — cinematic impact
  Heading             : "Rajdhani" or "Orbitron" — sci-fi / tech feel
  Body                : "IBM Plex Mono" or "Space Mono" — developer/terminal feel
  UI Elements         : "Chakra Petch" — futuristic labels

MOTION PRINCIPLES:
  - Scroll-driven: All major reveals triggered by ScrollTrigger
  - Ease: expo.out for enters, power2.in for exits
  - Duration: 0.6s–1.2s for hero elements; 0.3s–0.5s for UI elements
  - No bounce or elastic easing (too playful for Marvel tone)
  - Parallax: subtle on all background elements (0.2–0.4 factor)
  - Stagger: 0.08s between list item reveals

ICONOGRAPHY:
  - Custom SVG icons styled as Stark Industries interface glyphs
  - Tech stack: devicons.dev SVG set (colored versions)
  - UI icons: Lucide React or Phosphor Icons (thin weight)


--------------------------------------------------------------------------------
8. TECHNICAL ARCHITECTURE
--------------------------------------------------------------------------------

FRONTEND FRAMEWORK:
  - Next.js 14+ (App Router) — SSR/SSG for performance + SEO
  - TypeScript — strict mode

3D & ANIMATION:
  - Three.js / React Three Fiber — 3D scenes (hero, particle systems)
  - Drei — Three.js helpers (stars, floating text, environment)
  - GSAP + ScrollTrigger — scroll-driven animations, timelines
  - Framer Motion — component-level animations and page transitions
  - Lenis — smooth scroll inertia

STYLING:
  - Tailwind CSS — utility-first base
  - CSS Modules — component-specific complex styles
  - CSS Custom Properties — theme tokens

AI AGENT BACKEND:
  - Next.js API Routes (serverless) as proxy to Claude API
  - System prompt: injected developer knowledge base (JSON or .md file)
  - Streaming: Server-Sent Events for real-time response streaming

VOICE / TTS:
  - ElevenLabs API (preferred) or Web Speech API
  - Web Speech API for STT (user mic input)
  - Audio visualizer: Web Audio API AnalyserNode → Canvas

CONTACT FORM:
  - Resend or EmailJS for email delivery
  - React Hook Form + Zod for validation

HOSTING & DEPLOYMENT:
  - Vercel — automatic deployments from GitHub, edge functions
  - Domain: custom domain with HTTPS
  - Environment variables for all API keys

PERFORMANCE TARGETS:
  - LCP < 2.5s
  - FID < 100ms
  - CLS < 0.1
  - Lighthouse Desktop ≥ 90
  - Lighthouse Mobile ≥ 85
  - Three.js scene: stable 60fps on mid-range hardware

SEO:
  - Dynamic meta tags per section
  - Open Graph + Twitter Card for social sharing
  - JSON-LD schema (Person + WebSite)
  - Sitemap.xml
