Today's mission focused on enhancing the "Hero Dossier" (About Section) with high-end,

reacts to user interaction.

--------------------------------------------------------------------------------
KEY IMPLEMENTATIONS:
--------------------------------------------------------------------------------

1. GSAP-POWERED INTERACTIVE STAT BLOCKS
   - Component: `StatBlock.jsx`
   - Technical Achievement: Implemented GSAP ScrollTrigger to initiate number 
     count-up animations only when the stats enter the viewport.
   - Design: Styled with semi-transparent "void" backgrounds, holographic glows, 
     and decorative corner brackets to match the cinematic tech aesthetic.
   - Functionality: Automatically pads numbers (e.g., '05' instead of '5') and 
     supports custom suffixes like '⚡' or '☕'.

2. HOLOGRAPHIC BIOMETRIC PORTRAIT
   - Component: `HolographicPhoto.jsx`
   - AI Asset: Generated a cinematic developer portrait using custom prompts 
     mirroring the MCU lighting style (red/gold rim lighting).
   - Visual Effects:
       * Grayscale-to-Color: Image starts in grayscale and shifts to full color 
         on hover, simulating a "biometric identification" process.
       * Glitch Overlays: Subtle noise and color-shift animations on hover.
       * Scanlines: A continuous vertical scanline animation that "scans" the 
         subject's face.
       * HUD Overlays: Added holographic text (Coordinates, Clearance Level) 
         that reveals upon user interaction.

3. RESPONSIVE HUD LAYOUT
   - Integration: `AboutSection.jsx`
   - Desktop: Stats are positioned strategically around the photo frame using 
     absolute positioning, creating a layered, floating HUD feel.
   - Mobile: Stats transition into a clean, 2x2 grid for better readability 
     and touch interaction.

INTERVIEW PREPARATION — TECHNICAL TALKING POINTS:
--------------------------------------------------------------------------------

Q: How did you handle the count-up animations for the stats?
A: I used GSAP (GreenSock Animation Platform) because of its precision and 
ScrollTrigger plugin. Instead of a simple CSS transition, GSAP allows us to 
animate a raw JavaScript object value and update the React state on every frame 
using the `onUpdate` callback. This ensures the animation is perfectly smooth 
and synchronized with the scroll position.

Q: How did you implement the "Holographic" effect on the photo?
A: The effect is a combination of CSS filters and layered overlays. 
1. `grayscale(100%)` transition to `grayscale(0%)`.
2. A `mix-blend-mode: color` overlay for the blue tech tint.
3. A `radial-gradient` grid mask to give it that "screen" texture.
4. An infinite `@keyframes` animation for the scanline moving from `top: -2px` 
   to `top: 100%`.
5. Framer Motion for the smooth entry scale and rotation-Y (3D perspective) 
   animations.

Q: Why use GSAP instead of Framer Motion for the number counters?
A: While Framer Motion is excellent for layout and entry animations, GSAP is 
technically superior for "tweening" raw numbers and complex timeline sequencing. 
GSAP's `ScrollTrigger` also offers more granular control over the "start" and 
"end" markers, which is critical when you want multiple elements to trigger 
staggered animations as the user scrolls.

Q: How did you ensure performance with these animations?
A: 
- We used `gsap.context()` for clean-up, ensuring no memory leaks when 
  components unmount.
- The scanline and glitch effects use `pointer-events-none` to prevent them 
  from interfering with hover states.
- Images are optimized using `next/image` for automatic resizing and lazy loading.
- We used CSS animations for the simple repeating scanlines to keep the 
  JavaScript thread free for more complex interactions.
