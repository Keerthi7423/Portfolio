2. TECHNICAL CHALLENGES & SOLUTIONS
--------------------------------------------------------------------------------

CHALLENGE: Managing Smooth Transitions in a Custom Carousel
Standard React state changes can cause elements to "snap" in and out of existence.
SOLUTION: We used Framer Motion's `AnimatePresence` with `mode="wait"`. This 
ensures the "Exit" animation of the previous card completes before the "Enter" 
animation of the next card starts, creating a seamless transmission-wipe effect.

CHALLENGE: Creating a "Glitch" Effect Without Performance Loss
Heavy SVG filters or many DOM nodes can lag the browser during hover.
SOLUTION: We used a simple CSS-based approach. On hover, we trigger a state 
(`glitch: true`) that applies a `mix-blend-mode: screen` overlay and a 
`scale` pulse. This is GPU-accelerated and maintains a high frame rate.

CHALLENGE: Handling Timer Memory Leaks
`setInterval` in React can cause multiple timers to run if not cleaned up properly.
SOLUTION: We wrapped the auto-play logic in a `useEffect` and stored the 
interval ID in a `useRef`. The cleanup function `clearInterval(intervalRef.current)` 
ensures the timer is destroyed when the component unmounts or auto-play is disabled.

3. INTERVIEW PREPARATION (Q&A)
--------------------------------------------------------------------------------

Q: How did you build the carousel logic from scratch instead of using a library like Swiper?
A: I used a combination of React's `useState` for the index and Framer Motion's 
`AnimatePresence` for the animations. By using `key={activeIndex}` on the 
motion component, React treats each new index as a unique element, triggering 
the enter/exit animations automatically. This provides more control over the 
specific "glitch" and "blur" transitions than a standard slider library.

Q: Explain the "Manual Override" logic in your auto-play system.
A: I used a state variable `isAutoPlay`. When the component mounts, a `useEffect` 
starts an interval. If the user clicks "Next" or "Prev", I set `isAutoPlay` to 
false. This triggers the `useEffect` cleanup (clearing the interval) and 
prevents the carousel from jumping while the user is actively reading a specific 
testimonial.

Q: How do you handle image fallbacks in a UI that relies on specific avatars?
A: In the `TestimonialCard`, I implemented a stylized CSS fallback. If an 
image fails to load or is missing, a gradient background with the person's 
initials (generated via `.split(' ').map(n => n[0]).join('')`) is displayed 
within the holographic frame. This ensures the UI never looks "broken" even 
if assets are missing.

Q: Why use `mix-blend-mode` for the glitch effect?
A: `mix-blend-mode: screen` (or `overlay`) allows the glitch color to interact 
with the underlying avatar image dynamically. It mimics how light and color 
behave on a physical holographic display, making the effect feel more 
"integrated" than a simple semi-transparent color block.
